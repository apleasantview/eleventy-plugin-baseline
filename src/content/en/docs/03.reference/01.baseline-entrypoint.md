Reference for `_baseline/eleventy.config.js`: what the plugin wires up, the options you can pass, the defaults it exports, and what you still add yourself.

> Baseline setup has two parts: in your Eleventy config file, first use the config callback to add the plugin, then export your config object using Baseline’s defaults. See Eleventy’s “config shape” [docs](https://www.11ty.dev/docs/config-shapes/#callback-function).

The exported `config` object tells Eleventy where to look for input (`src`), where to write output (`dist`), which directories hold data and includes (`_data`, `_includes`, `assets`), and which template engines/formats to enable. Baseline exports these defaults so assets/head/sitemap work out of the box; start from them if you need to customize.

It’s recommended to keep Baseline’s default config object as a base because the modules are built around it. You can override it, but keep the shapes aligned so assets/head/sitemap continue to work. See [Baseline config defaults](/docs/reference/baseline-config-defaults/) for the full list and how to override safely.

## Default usage
```js
import baseline, { config as baselineConfig } from "@apleasantview/eleventy-plugin-baseline";

/** @param {import("@11ty/eleventy").UserConfig} eleventyConfig */
export default function (eleventyConfig) {
  eleventyConfig.addPlugin(baseline());
}

// Second configuration part.
export const config = baselineConfig;
```

## What it registers by default
- HtmlBasePlugin (uses `process.env.URL` or `pathPrefix`).
- Assets: assets-core, assets-postcss, assets-esbuild.
- Head: head-core.
- Sitemap: sitemap-core (template on by default).
- Navigator: navigator-core (template off by default).
- Debug filters: `_inspect`, `_json`, `_keys`.
- Filters: `markdownify`, `relatedPosts`, `isString`.
- Shortcodes: `image`.
- Dev server: `eleventyImageOnRequestDuringServePlugin`.
- Passthrough copy: `./src/static` → `/`.
- Draft preprocessor: skips pages with `draft: true` during `ELEVENTY_RUN_MODE=build`.

## Options (defaults + notes)

<div class="c-docs-steps">

- `verbose` (false) — emit extra logging from Baseline.
- `enableNavigatorTemplate` (false) — register the navigator template/routes; leave off unless you need the debug navigator.
- `enableSitemapTemplate` (true) — keep sitemap generation on by default; turn off if you manage sitemaps elsewhere.
- `multilingual` (false) — enable multilang core; requires `defaultLanguage` and a `languages` map.
- `defaultLanguage`, `languages` — required when `multilingual` is true; set your default IETF code and language map here.
- `assetsESBuild` ({ minify: true, target: "es2020" }) — forwarded to assets-esbuild; set `target`/`minify` to match your browser support and environment.

</div>

## Not included
- HTML image transform (`@11ty/eleventy-img` transform) is not bundled; add it separately if you want content-level `<img>` rewrites. Baseline’s image shortcode is available either way.

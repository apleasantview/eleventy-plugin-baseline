title: "Configuration"
---

Eleventy Baseline is a plugin you load in your `.eleventy.config.js`. It brings its own defaults but keeps room for your own config.

Right now, some wiring is opinionated/hard-wired (entrypoints, head tag, assets) while the project is still evolving; this is how it’s currently used day to day.

### Baseline plugin
In your root config:
```js
import baseline from "./_baseline/eleventy.config.js";
import { config as _config } from "./_baseline/eleventy.config.js";

export default function (eleventyConfig) {
  eleventyConfig.addPlugin(baseline({
    verbose: true,               // show extra logs
    Bob: "My number one guy.",   // example custom flag surfaced in _baseline
  }));
}

export const config = _config; // reuse Baseline’s dir/template defaults
```

### Directories
Baseline exports Eleventy dir defaults and creates a virtual directory for assets:
```
input: src
output: dist
data: _data
includes: _includes
assets: assets
```

### Head/meta defaults
`<baseline-head>` is injected via PostHTML. Defaults merge site/page data with `_data/head.js` (link/script) and set title/description/robots + OG/Twitter shells. You can extend via page/front matter `head:` or by editing `_data/head.js`.

### Site data
`_data/site.js` holds title/tagline/baseURL/defaultLanguage/noindex. Override per page in front matter as needed.

### Customizing further
You can still add your own filters, shortcodes, collections, and plugins in your root config. Baseline is meant to be additive, not exclusive.

I recommend creating an `_extend` directory, put all your customisations there and then import them like you are used to in your `.eleventy.config.js` file.
This keeps your overrides separate from Baseline’s files and makes future updates easier.

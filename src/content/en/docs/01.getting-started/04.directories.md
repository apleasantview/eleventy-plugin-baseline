## Overview
```shell
.
├── dist/
├── src/
│   ├── _data/
│   ├── _includes/
│   │   └── layouts/
│   ├── assets/
│   │   ├── css/
│   │   ├── js/
│   │   └── assets.11tydata.js
│   └── content/
│       ├── pages
│       └── posts
└── eleventy.config.js
```

- `eleventy.config.js`: root Eleventy configuration file.
- `dist/`: build output (generated output directory).
- `src/`: your site content, data, assets, and layouts aka input directory.

## Site source / Input directory (`src/`)
- `_data/`: global data (`site.js`, `head.js`, `helpers.js`).
- `_includes/`: layouts/partials (e.g., `layouts/base.njk`, `layouts/page.njk`).
- `assets/`: CSS (`assets/css/index.css`), JS (`assets/js/index.js`), images; excluded from collections via `assets.11tydata.js`.
- `content/`: pages/posts; locale folders (`en/`, `nl/`, `fr/`) hold localized content.
- `static/`: static resources such as favicon, robots.txt, ... . Passthrough copied to the root (`/`) of the site.

## About the assets directory (`assets/`)
The assets plugin creates a "virtual" `assets` dir in Eleventy’s config and exposes `assetsInput`/`assetsOutput` as global data.

You will need to manually create the `assets.11tydata.js` file with the following content:
```js
export default {
	eleventyExcludeFromCollections: true
}
```

## Site metadata (start here)
`_data/site.js` drives site-wide defaults for head/meta, sitemap, and languages. Example:
```js
export default {
  title: "Eleventy Plugin Baseline",
  tagline: "A magic carpet ride",
  url: process.env.URL,
  defaultLanguage: "en",
  noindex: false
}
```
- Set `url` for correct canonical/OG/sitemap URLs in production (or use the `siteUrl` option/env vars).
- Set `noindex: true` to suppress indexing and the sitemap.
- Set `defaultLanguage` for your pages; use locale folders for actual translations.

## Page metadata
- Set `title` and `description` in front matter. These feed head/meta defaults (and OG/Twitter shells via head-core).
- To noindex a page: `noindex: true` in front matter.

## Sitemap controls
- Per-page: `sitemap: { ignore: true, changefreq: "...", priority: 0.5, lastmod: "2024-01-01" }`
- Site-wide: `site.noindex = true` to skip the sitemap entirely.
- `lastmod` falls back to `item.date` if set; uses the `date.toUTCISO` global for formatting.

## Drafts
- Set `draft: true` to skip a page when `ELEVENTY_RUN_MODE=build`. Drafts still render in dev.

## Canonical and URLs
- Set `site.url` (or `siteUrl` option) and `pathPrefix` (if you deploy under a subpath) so canonical and OG URLs are correct. Falls back to `URL`/`DEPLOY_URL`/`DEPLOY_PRIME_URL` if set.

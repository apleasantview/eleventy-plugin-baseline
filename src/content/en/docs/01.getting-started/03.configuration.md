Eleventy Baseline is a plugin you load in your `eleventy.config.js`. It brings its own defaults but tries to keep room for your own config.

Right now, some wiring is opinionated/hard-wired (entrypoints, head tag, assets) while the project is still in its exploration phase; this is how it’s currently used day to day.

Additionally, the docs in `src` provide a good example of a standard config and site structure.

## Default configuration
This is what the current Baseline configuration looks like with defaults; add this to your root `eleventy.config.js` file:
```js
import baseline from "./_baseline/eleventy.config.js";
import { config as _config } from "./_baseline/eleventy.config.js";

eleventyConfig.addPlugin(baseline({
  verbose: false,                     // extra logs off by default
  enableNavigatorTemplate: false,     // navigator debug page off by default
  enableSitemapTemplate: true         // sitemap.xml on by default
}));

export const config = _config; // reuse Baseline’s dir/template defaults
```

## Extending your `eleventy.config.js` file
Add any custom code (filter, shortcode, plugin, collection, ...) after the Baseline plugin; just like any other Eleventy project.

### Tip
Keep your own customizations in an `_extend` folder next to your input folder (or inside your input directory) and import them in `eleventy.config.js` to simplify updates and keep your config file clean. [Read about structuring your configuration file.](https://www.lenesaile.com/en/blog/organizing-the-eleventy-config-file/)

## Config object
Baseline exports the Eleventy configuration object, setting directories defaults and a default virtual directory for assets. It also sets the default templating engines to Nunjucks and specifies template formats.

Default config object:
```js
export const config = {
  dir: {
    input: "src",
    output: "dist",
    data: "_data",
    includes: "_includes",
    assets: "assets"
  },
  htmlTemplateEngine: "njk",
  markdownTemplateEngine: "njk",
  templateFormats: ["html", "njk", "md"]
};
```
## Customize the config object
Provide your own config object if you feel the need to override the default `_config`.

Alternatively, you can do it inline at the end:
```js
export const config = {
  ..._config,
  dir: {
    ..._config.dir,
    output: "dist-site"
  }
};
```
That way you keep all baseline defaults and just extend what you need for your project.

## Other defaults Baseline sets
- Passthrough copy: `./src/static` → `/` (failOnError enabled).
- Drafts: pages with `draft: true` are skipped when `ELEVENTY_RUN_MODE=build`.
- Globals: `date.toUTCISO()` Nunjucks global for UTC ISO timestamps.

The Baseline plugin offers three debugging filters and two nunjuck globals to debug your templates.
## Debugging filters
- `_inspect(value, options={})`: pretty-prints any value using Nodeâ€™s `util.inspect` (pass depth/colors via `options`).
- `_json(value, space=0)`: `JSON.stringify` with optional indentation (defaults to no spaces).
- `_keys(value)`: returns the sorted keys of an object (useful to quickly see available properties).

### Filters usage
```js
{% raw %}{{ collections.all | _inspect() }}
{{ collections.all | _json }}
{{ page | _keys }}

// Inspect filter with options.
{{ collections.all | _inspect({getters: true, depth: 4, colors: true}) }}

// Advanced example. Key filter loop, show first item in array only.
{% for item in collections.all.slice(0, 1) %}
  {{ item | _keys }}
{% endfor %}{% endraw %}
```

## `navigator-core` Module
Sometimes we like to see the full picture. Use `_navigator()` or `_context()` in your templates and see internals and the whole data cascade. Provides an optional virtual template, lightly inspired by `phpinfo()`.

[Read more about the module.](/docs/modules/navigator-core/)

### Tips
- `_navigator()` and `_context()` in combination with the `_inspect()` filter can be a powerful debugging tool.
- If you use VSCode (or any other editor based on), consider [configuring the debugger](https://github.com/11ty/eleventy/issues/331#issuecomment-1559069278) to see passed values and more!

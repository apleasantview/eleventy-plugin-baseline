The Baseline plugin offers three debugging filters and two nunjuck globals to debug your templates.
## Debugging filters
- `inspect(value, options={})`: pretty-prints any value using Node’s `util.inspect` (pass depth/colors via `options`).
- `json(value, space=0)`: `JSON.stringify` with optional indentation (defaults to no spaces).
- `keys(value)`: returns the sorted keys of an object (useful to quickly see available properties).

### Filters usage
```html
{% raw %}{{ collections.all | inspect({getters: true, depth: 4, colors: true}) }}{% endraw %} // inspect with options
{% raw %}{{ collections.all | json }}{% endraw %} // json output
{% raw %}{{ page | keys }}{% endraw %} // keys from page object

// Advanced keys filter examples. Returns first item in array.
{% raw %}{% for item in collections.all.slice(0, 1) %}{% endraw %}
{% raw %}  {{ item | keys }}{% endraw %}
{% raw %}{% enfor %}{% endraw %}
```

## `navigator-core` Module
### What it does
- Adds a Nunjucks global `_navigator` to inspect the full render context (handy for debugging).
- When `enableNavigatorTemplate` is true, registers a virtual page at `/navigator-core.html`.

{% alertBlock %}
Curious? [See the Navigator template in action](/navigator-core.html).
{% endalertBlock %}

### Configuration defaults
```js
eleventyConfig.addPlugin(baseline({ enableNavigatorTemplate: false }));
```

### How it works
- Registers `_navigator()` to return the current Nunjucks context.
- Registers `_context()` — a child of `_navigator()`.
- If `enableNavigatorTemplate: true`, ads the bundled `navigator-core.html` template.

### Example usage
```njk
{% raw %}{% for key, value in _navigator() %}{% endraw %}
	<details>
		<summary><strong>{% raw %}{{ key }}{% endraw %}</strong></summary>
		{% raw %}{% if value | isString %}
			{% highlight "js" %}
				{{ value | safe }}
			{% endhighlight %}
		{% else %}
			{% highlight "js" %}
				{{ value | inspect | safe }}
			{% endhighlight %}
		{% endif %}{% endraw %}
	</details>
{% raw %}{% endfor %}{% endraw %}

```

### Tips
- `_navigator()` and `_context()` in combination with the `inspect()` filter can be a powerful debugging tool.
- You can set the depth of the inspector by using a tuple: `baseline({ enableNavigatorTemplate: [true, 4] })`
- If you use VSCode (or any other editor based on), consider [configuring the debugger](https://github.com/11ty/eleventy/issues/331#issuecomment-1559069278) to see passed values and more!

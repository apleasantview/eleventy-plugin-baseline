Here’s how Baseline is laid out today. It’s intentionally simple while things evolve.

## Top-level
- `.eleventy.config.js`: loads the Baseline plugin and re-exports its dir/template defaults.
- `_baseline/`: the Baseline plugin code (filters, shortcodes, modules).
- `src/`: your site content, data, assets, and layouts.
- `dist/`: build output (generated).
- `temp/`: scratch/export (ignored).

### Baseline internals (`_baseline/`)
- `eleventy.config.js`: the Baseline plugin factory.
- `modules/`: feature modules:
  - `assets-core`, `assets-postcss`, `assets-esbuild`
  - `navigator-core`, `navigator-context`
  - `head-core`
  - `feature-i18n` (stub), `seo-matter` (stub)
- `filters/`, `shortcodes/`, `debug.js`, `helpers.js`

### Site source (`src/`)
- `_data/`: global data (`site.js`, `head.js`, `helpers.js`, `possums.json`).
- `_includes/`: layouts/partials (e.g., `layouts/base.njk`, `layouts/page.njk`).
- `assets/`: CSS (`assets/css/index.css`), JS (`assets/js/index.js`), images; excluded from collections via `assets.11tydata.js`.
- `content/`: pages/posts; docs live in `content/en/docs/`; locale folders (`en/`, `nl/`, `fr/`) hold localized content.

### Virtual assets directory
The assets plugin creates a virtual `assets` dir in Eleventy’s config and exposes `assetsDir`/`assetsDirRelative` as global data. It also adds `assetUrl` filter/shortcode for URL generation and a collection of assets; passthrough is optional.

### Head and layout
- Head is injected with `<baseline-head>` (PostHTML) using site/page data plus `_data/head.js`.
- Base layout: `src/_includes/layouts/base.njk` renders `<baseline-head>`, header, main content block, and build info.

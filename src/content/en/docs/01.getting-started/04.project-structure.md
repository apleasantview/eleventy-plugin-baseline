## Overview
```bash
.
├── dist/
├── src/
│   ├── _data/
│   │   └── site.js
│   ├── _includes/
│   │   └── layouts/
│   ├── assets/
│   │   ├── css/
│   │   │   └── index.css
│   │   ├── js/
│   │   │   └── index.js
│   │   └── assets.11tydata.js
│   └── content/
│       ├── pages
│       └── posts
└── eleventy.config.js
```

- `eleventy.config.js`: root Eleventy configuration file.
- `dist/`: build output (generated output directory).
- `src/`: your site content, data, assets, and layouts aka input directory.

## Site source / Input directory (`src/`)
- `_data/`: global data (`site.js`, `head.js`, `languages.js`, helpers).
- `_includes/`: layouts/partials/components (e.g., `layouts/base.njk`, `components/docs-nav.njk`).
- `assets/`: CSS (`assets/css/index.css`), JS (`assets/js/index.js`), images; excluded from collections via `assets.11tydata.js`.
- `content/`: posts/pages, locale folders (`en/`, `nl/`, `fr/`) if multilingual project.
- `static/` (if present): static resources passthrough-copied to `/`.

## About the assets directory (`assets/`)
The assets plugin creates a virtual `assets` dir in Eleventy’s config and exposes the resolved paths via `_baseline.assets`:
```
_baseline.assets.input   // e.g., ./src/assets/
_baseline.assets.output  // e.g., ./dist/assets/
```

Create an `assets.11tydata.js` directory data file in `assets/` to exclude assets from collections:
```js
export default {
  eleventyExcludeFromCollections: true
}
```

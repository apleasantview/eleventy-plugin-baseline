title: "Multilang Site"
---

## What it does
Baseline can wire Eleventy’s i18n plugin (via multilang-core) and generate per-language sitemaps (via sitemap-core) when multilingual is enabled.

## Configure languages
Create `src/_data/i18n.js` with your locales. Example:
```js
export default {
  defaultLanguage: "en",
  languages: {
    en: { contentDir: "content/en/", languageCode: "en", languageName: "English", title: "Eleventy Plugin Baseline", tagline: "A magic carpet ride" },
    nl: { contentDir: "content/nl/", languageCode: "nl", languageName: "Nederlands", title: "Eleventy Plugin Baseline", tagline: "Een rit op een magisch tapijt" },
    fr: { contentDir: "content/fr/", languageCode: "fr", languageName: "Français", title: "Eleventy Plugin Baseline", tagline: "Un tour en tapis volant" }
  }
};
```
- `defaultLanguage` is required for fallbacks.
- `languages` has to be an object (as above); Baseline normalizes it to decide if multilingual is on and to pass to sitemap-core.

## Enable multilang in Baseline
In your root `eleventy.config.js`:
```js
import i18n from "./src/_data/i18n.js";
import baseline from "./_baseline/eleventy.config.js";
import { config as _config } from "./_baseline/eleventy.config.js";

export default function (eleventyConfig) {
  eleventyConfig.addPlugin(baseline({
    multilingual: true,
    defaultLanguage: i18n.defaultLanguage,
    languages: i18n.languages
  }));
}

export const config = _config;
```

## Authoring content
- Organize content by locale folders under `src/content/` (e.g., `en/`, `nl/`, `fr/`).
- for each locale folder, create a directory data file, e.g. `en/`:
  ```js
  // filename: en.11tydata.js
  export default {
    lang: "en"
  }
  ```
- Each localized page should set:
  ```yaml
  lang: en           # or nl, fr...
  translationKey: my-page
  title: "My Page"
  ```
- Keep a default-language version present; it powers `x-default` and fallbacks.

## Templates and alternates
- `collections.translations` (from multilang-core) maps `translationKey` → `{ langCode: { title, url, lang, isDefault, data } }`.
- Use it to render hreflang/alternate links; sitemap-core will emit `<xhtml:link>` alternates automatically when translations exist.

## Sitemaps
- With `multilingual: true` and multiple languages, sitemap-core emits per-language sitemaps at `/lang/sitemap.xml` plus a sitemap index at `/sitemap.xml`.
- If you want single-language only, set `multilingual: false` or provide a single language.

## Tips
- Set `site.defaultLanguage` in `_data/site.js` to match your `defaultLanguage`.
- If you deploy under a subpath, set `pathPrefix`; set `site.url` (or `siteUrl` option/env vars) for correct canonical/sitemap URLs.
- For labels (e.g., nav), store strings in a per-language map and pick by `page.lang` or `site.defaultLanguage`.

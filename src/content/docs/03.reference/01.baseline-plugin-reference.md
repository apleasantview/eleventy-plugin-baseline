Reference walkthrough of the Baseline plugin. This mirrors `@apleasantview/eleventy-plugin-baseline/eleventy.config.js` in code order.

## Purpose

Defines the Baseline plugin (options, modules, filters, shortcodes) and exports the Eleventy config defaults (`baselineConfig`).

## Version check and options

- `versionCheck(">=3.0")` warns if Eleventy is too old.
- `hasImageTransformPlugin` detected from `eleventyConfig.hasPlugin()` — not a user option, resolved before options are built.
- `userOptions` built from explicit picks — each key has a default:
  - `verbose` (false), `enableNavigatorTemplate` (false), `enableSitemapTemplate` (true)
  - `assets.esbuild` (empty object — esbuild defaults apply inside the module)
  - `multilingual` (false), `defaultLanguage`, `languages`
- Multilang guard: requires `multilingual: true`, a truthy `defaultLanguage`, and a non-empty `languages` map. If any are missing, multilang-core doesn't load.

## Globals, passthrough, drafts

- Adds global `_baseline` — a curated object with `version`, `name`, `verbose`, `hasImageTransformPlugin`. See the [globals reference](/docs/globals/) for the full shape.
- Passthrough copy: `./src/static` → `/`.
- Draft preprocessor: skips `draft: true` pages when `ELEVENTY_RUN_MODE === "build"`. Guarded against double-registration — if you define your own drafts preprocessor, Baseline won't overwrite it.

## Modules registered

Registration order matters — listed here in the order they appear in the entry point:

1. `multilang-core` (conditional) — language collections, hreflang, i18n filters. Only when `multilingual: true` with valid config.
2. `HtmlBasePlugin` — `baseHref` from `process.env.URL` or `pathPrefix`.
3. `assets-core` — asset pipeline orchestrator. Adds `_baseline.assets` to global data.
4. `head-core` — enables `<baseline-head>` for HTML head injection.
5. `sitemap-core` — XML sitemap generation (multilingual-aware).

## Filters, shortcodes, debug

- Filters: `markdownify`, `relatedPosts`, `isString`.
- Shortcode: `image`.
- Debug filters: `_inspect`, `_json`, `_keys`.
- `navigator-core` — debug globals (`_navigator`, `_context`) and optional virtual page. Registered last.

## Dev server

- Adds `eleventyImageOnRequestDuringServePlugin` for on-request image generation during serve.

## Plugin metadata

- Sets `plugin.name` to `@apleasantview/eleventy-plugin-baseline` so `eleventyConfig.hasPlugin()` can detect it.

## Exported config defaults

```js
export const config = {
	dir: {
		input: 'src',
		output: 'dist',
		data: '_data',
		includes: '_includes',
		assets: 'assets'
	},
	htmlTemplateEngine: 'njk',
	markdownTemplateEngine: 'njk',
	templateFormats: ['html', 'njk', 'md']
};
```

## Not included

- HTML image transform plugin (`eleventyImageTransformPlugin`) wiring — add it yourself if you need content-level `<img>` rewrites.

## Customize safely

Start from `baselineConfig` and merge if you change dirs, engines, or formats, so assets, head, and sitemap modules continue to work.

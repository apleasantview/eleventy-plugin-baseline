Baseline ships as five feature modules. Each one wires a distinct concern into Eleventy and stays out of the way of the others. The pages below cover what each module does, when it activates, and the options it accepts.

If you are starting out, the [concepts page](/docs/introduction/concepts/) introduces the three-layer model these modules sit on top of. The [tutorials](/docs/tutorial/) walk through the common pairings.

## The five modules

- **[assets](/docs/modules/assets/)**: wires JS (esbuild) and CSS (PostCSS) entry points into Eleventy, with inline filters for critical-path assets.
- **[head](/docs/modules/head/)**: replaces a `<baseline-head>` placeholder with a sorted, deduped element list at transform-time.
- **[multilang](/docs/modules/multilang/)**: language normalisation, per-page `locale`, translation map, and the i18n filters. Opt-in.
- **[navigator](/docs/modules/navigator/)**: debug surface. Globals, filters, and a virtual page for inspecting render state. On in dev by default.
- **[sitemap](/docs/modules/sitemap/)**: a single `/sitemap.xml`, or per-language sitemaps plus an index when multilang is active.
